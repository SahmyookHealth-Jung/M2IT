##### 5단계 설계 프로세스 

1. 상태 정의 
   => 내부적으로 사용하는 핵심 데이터. 
   => 코드가 시작될 때 정의된다.
   EX) private int currentScore = 0; // 점수가 얼마인지 기억하는 상태
   EX) private bool isHomeTeamSelected = true; // 현재 선택된 팀이 홈팀인지 아닌지 기억하는 상태 
   
2. 이벤트 정의
   => 사용자의 행동(클릭, 입력)과 코드를 연결하는 과정.
   => **코드를 직접 실행하는 것이 아니라, '행동이 발생하면 이 함수를 실행하라'고 등록하는 것**

3. 로직 구현
   => 이벤트가 발생했을 때, 실제로 상태를 변경하는 핵심 함수.
   => 이 함수는 외부의 표현(UI)가 아니라 내부의 상태 변수만 건드린다.
   EX) 

   private void HandleScoreChange(int amount)
   {
   // 1. 내부 상태만 변경 (currentScore)
   currentScore += amount; 

   // 2. 외부 표현(UI)을 건드리지 않고, 5단계의 함수를 호출합니다.
   UpdateScoreDisplay(); 
   }
   
4. 경계 조건 처리
   => 입력이나 상태의 한계 방어
