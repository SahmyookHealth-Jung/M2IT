C# 데이터 저장



* 배열

=> 여러 개의 값을 저장할 수 있는 데이터 형 중 하나

=> 데이터형\[] 변수명; (배열 선언하기)  -> 이 상태는 값이 들어가지 않은 상자만 만들어둔 것과 같습니다.



* 컬렉션(Collections)

=> 컬렉션은 단일 자료 구조가 아닌 입력과 출력, 처리를 포함한 연산을 수행하는 다양한 자료 구조의 모음

=> ArrayList, Queue, Stack, Hashtable 등이 존재

=> 컬렉션 클래스는 메소드를 통해 데이터를 다룹니다.



\*\* ArrayList

=> 배열과 유사한 데이터 묶음을 관리하는 클래스

=> 크기를 미리 설정할 필요가 없고, 한 가지 데이터 형이 아닌 여러 다양한 데이터 형의 값을 담을 수 있습니다.

=> Add, Remove 메소드를 통해 추가 및 제거를 할 수 있습니다.

=> var 형을 사용해 암시적으로 형변환을 할 수 있도록 합니다.



\*\* Queue 클래스 

=> 입력한 값이 먼저 출력되는 자료 구조

=> 선입선출

=> Enqueue - 추가 / Dequeue - 삭제



\*\* Stack 클래스

=> 먼저 입력한 데이터가 맨 나중에 출력되는 자료 구조

=> push / pop



\*\* Hashtable 클래스

=> 키(Key)와 값(Value)으로 저장하는 자료 구조입니다.

=> 데이터의 크기가 증가할수록 처리하는 속도가 빠르다는 장점

=> 데이터를 key 값으로 한 번에 찾을 수 있기때문에 데이터가 커질수록 유리합니다.



* 예외처리

=> EX) 나누기를 할 때, 0으로 나누면 오류 메시지가 출력되고 프로그램이 강제 종료됩니다.

=> try - catch 사용하기오후 5:32 2025-10-15



\*\* e.Message 와 e.ToString() 의 차이는 출력되는 정보의 양과 형식이다.

-> e.Message의 경우 간결한 설명(사용자에게 문제가 무엇인지 알려줄때)

-> e.ToString()의 경우 자세한 설명(개발자가 디버깅할 때 예외의 경로를 추적할 때)
