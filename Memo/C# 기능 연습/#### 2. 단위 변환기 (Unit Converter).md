#### **#### \*\*2. 단위 변환기 (Unit Converter)\*\***

배우는 핵심 개념



\* 입력 유효성 검사: 숫자가 아닌 문자가 입력되지 않도록 TextBox의 입력을 확인하고 경고하는 방법을 배웁니다.

\* 공식 적용 및 데이터 타입: int 대신 double이나 decimal을 사용하여 소수점 계산을 정확하게 처리하는 방법을 익힙니다.

\* UI 동기화: 입력 창이 변경될 때마다 결과가 실시간으로 업데이트되도록 이벤트를 연결합니다.



\*\*주요 기능\*\*



마일(Mile) 입력 TextBox

킬로미터(Km) 입력 TextBox

변환 공식 : Mile\_To\_Km\_Factor = 1.60934M;



\# decimal.TryParse(string s, out decimal result)

\- 만약 유효한 숫자라면, 그 숫자로 변환된 값을 두 번째 매개변수 (out inputValue)에 '담아서' 메서드 밖으로 내보냅니다.

 

섭씨(Celsius) 입력 TextBox

화씨(Fahrenheit) 출력 TextBox

변환 공식: $\\\\text{F} = \\\\text{C} \\\\times 1.8 + 32$

역할 : 유효한 숫자라면, 그 숫자로 변환된 값을 두 번째 매개변수 out inputValue에 담아서 메서드 밖으로 내보냅니다.

* 순환 호출 방지 ($\\text{isChangingText}$): $\\text{TextBox}$ 두 개가 서로의 값을 바꾸면서 무한 루프에 빠지는 것을 막는 것은 전문가 수준의 디버깅 기술이 필요합니다.
* 양방향 상태 관리 ($\\text{isMileInputMode}$): 입력 필드 두 개 중 하나만 활성화하는 복잡한 UI 상태 관리입니다.
* 실시간 유효성 검사 ($\\text{TryParse}$): 키를 누르는 순간마다 계산하고 오류를 표시하는 정교함이 요구됩니다.
