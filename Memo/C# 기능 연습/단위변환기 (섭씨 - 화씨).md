### 단위변환기 (섭씨 -> 화씨)



섭씨(Celsius) 입력 TextBox

화씨(Fahrenheit) 출력 TextBox

변환 공식: $\\\\text{F} = \\\\text{C} \\\\times 1.8 + 32$



* mile 변환기에서는 class 레벨에 inputValue를 두었지만, 섭씨/화씨 변환기에서는 ConvertValue 메서드 안에 demical celsius를 선언한 것은 해당 값이 앱의 '상태'가 아니기 때문이다.

=> WinForms(윈폼)에서 상태와 지역 변수를 구분하는 핵심 원칙.



* 섭씨/화씨 앱의 '상태'

=> 계산기 / 점수판의 상태 : 계산을 완료하기 위해 이벤트 간의 값을 기억해야 한다.

=> 섭씨 / 화씨 앱의 상태 : 앱이 유일하게 기억해야 할 것은 변환 공식 상수! 사용자가 입력한 현재 섭씨 값은 다음 이벤트를 위해 기억할 필요가 없음. 입력이 들어오는 순간 계산하고 출력하면 역할이 끝남.
