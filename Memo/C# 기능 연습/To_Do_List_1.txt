구조 분석

- 코드의 3가지 기둥

기둥 1. 데이터 모델 
public class TodoItem {} 
=> 무엇을 저장할지 정의한다. 현실의 '할 일 항목'을 프로그램 세계로 가져온 설계도.
=> {get; set;} 으로 정의된 속성만 가지며, 이 속성들이 DataGridView의 컬럼과 연결.
=> public bool Completed {get; set;} (완료 여부) / public string TaskDescription {get;set;} (할 일 내용)
=> 내가 만들 앱이 가장 중요한 정보를 어떻게 저장해야 할까 ? 

기둥 2. 데이터 컨테이너 
BindingList<TodoItem> todoItems = new BindingList<TodoItem>();
=> TodoItem 객체들을 실제로 저장하고, DataGridView와 자동으로 연결하는 중간 다리 역할입니다.
=> todoItems 변수에 객체를 추가(Add)하거나 제거(RemoveAt)하면, dgvTask.DataSource 덕분에 UI가 자동으로 새로고침 됩니다.
dgvTask.DataSource = todoItems;

기둥 3. 기능 로직 
private void btnAdd_Click() {}
=> 사용자의 행동에 반응하여 데이터를 조작
=> TextBox에서 입력값을 읽어오고 새 TodoItem을 생성하고 todoItems 에 추가합니다. for 루프와 조건문으로 완료된 항목을 역순으로 순회하며 삭제합니다.

B. 장기적인 '창조' 학습 습관
변수 역할 정의: 새로운 코드를 작성할 때, 변수나 클래스를 만들기 전에 **"이 변수는 뭘 저장할까?"**라고 스스로 질문하고 메모하세요. (예: todoItems는 TodoItem 객체를 담는 창고다.)

이벤트 정의부터 시작: 기능 구현 시, **어떤 행동(이벤트)**이 필요한지 먼저 정의하세요. (예: 항목 추가는 "버튼 클릭" 이벤트가 필요하다.)

복잡한 로직은 순서도: 삭제 로직처럼 복잡한 for 루프가 필요할 때는 코드를 작성하기 전에 종이에 **1번부터 5번까지의 순서(순서도)**를 적어보세요. (예: 1. 리스트를 끝에서부터 본다. 2. Completed가 True인지 확인한다. 3. True면 제거한다. 4. 다음 항목으로 넘어간다.)
