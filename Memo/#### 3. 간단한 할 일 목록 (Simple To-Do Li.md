\#### \*\*3. 간단한 할 일 목록 (Simple To-Do List)\*\*



배우는 핵심 개념



* 리스트(List) 상태 관리: 단순히 하나의 숫자 변수가 아닌, List<string> 같은 컬렉션을 상태로 관리하고 조작하는 방법을 배웁니다.
* 
* UI와 데이터 동기화: 사용자가 항목을 추가/삭제할 때마다 ListBox와 내부 List<string> 데이터를 일치시키는 방법을 연습합니다.



주요 기능

* &nbsp;새로운 할 일을 입력하는 TextBox
* &nbsp;항목을 추가하는 버튼 (Add)
* &nbsp;할 일 목록을 표시하는 ListBox
* &nbsp;선택된 항목을 목록에서 제거하는 버튼 (Remove)



핵심 개념 이해 

* 데이터 바인딩

=> UI(DataGridVeiw)가 데이터를 저장하는 변수(taskList)와 연결되어, 변수만 바꾸면 UI가 자동으로 업데이트되는 기능 

=> BindingList<T>		

* 이벤트 처리

=> 사용자의 행동 (버튼 클릭, 체크박스 클릭 등)에 대응하여 특정 메서드를 실행하는 것

=> btnAdd\_Click , CellContentClick

* 객체 지향 모델링

=> 현실의 '할 일 항목'을 프로그램 속의 클래스(TodoItem)로 정의하고, 그 속성(Completed, TaskDescription)을 활용하는 것.
